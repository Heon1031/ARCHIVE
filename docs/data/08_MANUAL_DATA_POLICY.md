# 수기 입력 데이터 정책

## 기본 원칙

수기 입력 데이터는 API 우선 흐름을 보완하는 보조 데이터다.

## 사용 조건

- Brunch
- Blog
- API 미지원 플랫폼
- API 실패 콘텐츠
- 일시적으로 데이터를 불러오지 못한 콘텐츠

## 저장 방식

- 수기 콘텐츠도 `ContentItem` 구조로 저장한다.
- 수기 성과도 `InsightRecord` 구조로 저장한다.
- `source: "manual"`을 반드시 사용한다.
- 수기 성과는 `apiSyncStatus: "manual"`로 저장한다.

## 필드 기준

- 계정 또는 출처는 `Account`로 등록한다.
- API 미지원 플랫폼은 `isApiSupported: false`로 표시한다.
- 수기 데이터도 `accountId`와 `contentId` 연결 기준을 유지한다.

## UI 표시 기준

- 수기 데이터는 API 데이터와 출처를 구분해 보여준다.
- 수기 입력 버튼은 보조 액션으로 표시한다.
- 수기 데이터를 전체 성과의 핵심 근거처럼 강조하지 않는다.

## 금지 사항

- 수기 입력을 기본 콘텐츠 등록 방식으로 만들지 않는다.
- API 실패를 수기 데이터로 조용히 덮어쓰지 않는다.
- 수기 데이터와 API 데이터를 같은 신뢰도로 표현하지 않는다.
