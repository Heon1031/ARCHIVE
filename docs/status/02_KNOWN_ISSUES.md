# 알려진 이슈

## 보안

- Access Token은 MVP에서 `localStorage`에 저장된다.
- 이 저장 방식은 개인 테스트용이며 실제 배포용 보안 구조가 아니다.
- 공개 배포 전에는 백엔드, 토큰 보관, 권한 분리, API 프록시 구조가 필요하다.
- App Secret과 실제 계정 비밀값은 문서나 프론트 코드에 기록하지 않는다.

## API

- Instagram / Threads endpoint와 metric 이름은 실제 계정 테스트와 Meta 공식 문서 기준으로 재검증해야 한다.
- 플랫폼별로 제공하지 않는 지표는 `undefined`로 다룬다.
- API 실패 메시지는 실제 응답을 보며 추가 보정이 필요할 수 있다.

## 프론트 구조

- 계정 설정 탭에 API 상세 기능이 많이 모여 있어 추후 추가 축소 여지가 있다.
- 수기 입력은 보조 기능으로만 유지해야 하며 메인 작업처럼 강조하면 안 된다.

## 제외 사항

- App Secret은 프론트에 넣지 않는다.
- OAuth 전체 플로우는 현재 MVP 범위가 아니다.
- 자동 업로드는 현재 MVP 범위가 아니다.
