## 2. 완료된 작업

V 1단계: 리마인드 구조 복구
- today-reminder-card를 operation-panel 내부로 이동했다.
- 가로모드에서도 리마인드가 보이게 했다.
- 세로모드에서도 리마인드가 유지되게 했다.
- docs/design/06_LAYOUT_WIREFRAMES.md에 Operation Panel 기준을 보강했다.

V 1.5단계: 리마인드 UX / 멀티유즈 후보 추가
- 리마인드 제목을 짧게 정리했다.
- 리마인드를 썸네일 중심 카드로 바꿨다.
- 이전/다음 버튼을 제거했다.
- 점 indicator로 후보를 이동하게 했다.
- 자동 슬라이드를 추가했다.
- 멀티유즈 후보 카드를 Operation Panel에 추가했다.

V 1.7단계: 카드 균형 / 리마인드 미세 정리
- 세로모드 Operation Panel 카드 균형을 정리했다.
- 리마인드 후보 개수 pill을 제거했다.
- 리마인드 점 indicator를 중앙 정렬했다.
- 리마인드 버튼/컨트롤 중앙 정렬을 체크리스트에 반영했다.
- 자동 슬라이드를 8초 기준으로 조정했다.
- 리마인드 modal에 max-height와 overflow-y를 적용했다.
- modal 본문 line-clamp 문제를 보정했다.

V 2단계: 운영 판단 상세 정리
- 추천 뱃지 클릭 시 추천 상세만 보이도록 정리했다.
- 실제 게시물 클릭 시 게시물 결과와 인사이트 요약이 보이도록 분리했다.
- 개선 후보 클릭 시 개선 대상, 낮은 지표, 개선 이유, 다음 보완점이 보이도록 정리했다.
- 상세 카드 문구를 짧게 줄이고 중복 설명을 줄였다.
- modal에서는 본문이 잘리지 않도록 CSS를 보강했다.

V 2.4단계: Graphify 기반 MainTab 분리 계획 문서화
- Graphify 결과에서 MainTab.tsx가 54 edges, MainTab()이 15 edges를 가진 허브임을 확인했다.
- MainTab의 달력, 운영 판단, 리마인드, 멀티유즈, 키워드, 상세 선택 책임을 문서로 분리했다.
- `docs/status/06_MAINTAB_SPLIT_PLAN.md`에 안전한 최소 분리 순서를 기록했다.

V 2.5단계: 개선 후보 목록 / 추천 키워드 근거 / 분포 bar
- 개선 후보 카드 클릭 시 후보 목록 modal이 열리도록 했다.
- 후보 목록에서 게시물을 선택하면 하단 상세가 개선 후보 상세로 바뀌게 했다.
- 추천 키워드 카드 클릭 시 추천 근거 modal이 열리도록 했다.
- 추천 키워드 modal에 이번 달 사용 횟수, 최근 반복 키워드, 부족한 키워드, 관련 과거 게시물을 표시했다.
- 키워드별 사용 빈도는 CSS 기반 bar로 표시했다.

V 2.6단계: 추천 키워드 분포 시각화 고도화
- 추천 키워드 modal 상단에 추천 키워드와 한 줄 작성 방향을 명확히 배치했다.
- 추천 근거를 3개 이내의 짧은 목록으로 정리했다.
- CSS 기반 키워드 분포 bar의 간격, 최소 폭, 추천 키워드 강조를 보강했다.
- 관련 과거 게시물은 제목, 1줄 요약, 날짜/플랫폼/형식만 보이도록 정리했다.

V 3단계: 달력 UX 정리
- 실제 게시물 뱃지를 플랫폼+형식 기준으로 표시했다.
- 추천/휴식 뱃지 색상과 hover/cursor 상태를 정리했다.
