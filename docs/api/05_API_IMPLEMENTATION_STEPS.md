# API 구현 단계

8단계는 한 번에 크게 구현하지 않고 작은 단위로 나눈다. 각 단계는 계획, 구현, QA, 커밋 순서로 진행한다.

## 권장 순서

1. API 문서/보안 범위 확정
2. 계정 설정 탭의 입력값 점검
3. Instagram 연결 테스트
4. Threads 연결 테스트
5. 최근 게시물 목록 조회
6. 게시물 인사이트 조회
7. `InsightRecord` 생성/저장 연결
8. API 실패 시 수기 입력 보조 연결
9. API 기능 QA와 축소

## 진행 원칙

- Instagram과 Threads를 한 커밋에 과하게 섞지 않는다.
- 연결 확인, 게시물 조회, 인사이트 조회를 분리해 검토한다.
- API 원본 응답은 서비스 계층에서 정규화한다.
- UI 컴포넌트 안에 API 세부 로직을 길게 작성하지 않는다.
- 실제 endpoint, metric, 권한 이름은 구현 직전에 최신 공식 문서로 확인한다.
