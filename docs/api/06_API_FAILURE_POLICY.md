# API 실패 처리 정책

## 실패 유형

- 토큰 없음
- 토큰 만료
- 권한 부족
- 계정 ID 오류
- 미디어 또는 Post ID 오류
- API 응답 형식 변경
- 네트워크 오류
- rate limit 가능성

## 기본 원칙

- API 실패로 앱 전체가 멈추면 안 된다.
- 실패한 계정이나 콘텐츠만 명확히 표시한다.
- 사용자가 다음 행동을 알 수 있는 짧은 메시지를 보여준다.
- 토큰 원문이나 민감한 응답은 에러 메시지에 포함하지 않는다.

## 기록 원칙

- `apiSyncStatus`로 성공, 실패, 만료, 수기 입력 상태를 구분한다.
- `syncErrorMessage`에는 사용자에게 필요한 최소 원인만 저장한다.
- `lastSyncedAt`은 마지막 동기화 시도 또는 성공 시간을 기록한다.
- 실패 기록은 `InsightRecord` 또는 계정 상태와 연결해 추적한다.

## 수기 입력 보조 전환

API 실패 콘텐츠는 수기 입력 보조 흐름으로 전환할 수 있다. 단, 수기 입력은 API를 대체하는 메인 기능이 아니라 실패 케이스와 API 미지원 플랫폼을 위한 보조 기능으로 표시한다.
