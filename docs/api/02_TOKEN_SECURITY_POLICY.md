# 토큰 보안 정책

## 기본 원칙

- Access Token은 계정 등록/설정 탭에서만 입력하고 관리한다.
- Access Token 원문은 메인 탭, 성과 탭, 계정 카드에 그대로 노출하지 않는다.
- App Secret은 프론트 코드, localStorage, UI 입력 폼에 절대 넣지 않는다.
- `.env`, `.env.local`은 이번 MVP에서 생성하지 않으며 Git에 커밋하지 않는다.

## MVP localStorage 한계

MVP는 개인 테스트용으로 localStorage 저장을 허용한다. 다만 localStorage는 브라우저에서 접근 가능하므로 실제 배포용 보안 저장소가 아니다.

## 개인용 MVP와 배포용 구조 차이

- 개인용 MVP: 사용자가 직접 입력한 토큰을 브라우저에 저장하고 테스트한다.
- 실제 배포: 백엔드가 토큰을 암호화해 저장하고, 프론트는 백엔드 API만 호출한다.
- 실제 배포: App Secret과 토큰 갱신 로직은 서버에서만 처리한다.

## 백엔드 프록시가 필요한 조건

- App Secret이 필요한 API 흐름
- 토큰 갱신 또는 장기 토큰 관리
- 여러 사용자 계정 지원
- 배포 환경에서 토큰을 안전하게 숨겨야 하는 경우
- API 호출 로그, rate limit, 실패 재시도 정책이 필요한 경우

## UI 노출 기준

- 토큰은 `입력됨`, `미입력`, 일부 마스킹 수준으로만 표시한다.
- 에러 메시지에 토큰 원문을 포함하지 않는다.
- 계정 ID는 계정 설정 탭에서만 다루고 메인 작업 화면에는 노출하지 않는다.
