# UX 개요

## 전체 방향

사용자가 앱을 열었을 때 오늘 해야 할 콘텐츠 운영 작업을 먼저 파악하고, 필요할 때 성과와 계정 연결 상태를 분리해서 확인할 수 있어야 한다.

## 탭별 역할

- 메인 탭: 콘텐츠 캘린더, 기획, 제작 상태, 리마인드, 오늘 할 일
- 성과 탭: 전체/플랫폼별/계정별/콘텐츠별 성과 확인
- 계정 등록·설정 탭: 계정 ID, Access Token, 연결 상태, 마지막 동기화, API 테스트 관리

## 사용자 주요 행동

- 계정을 등록하고 API 연결 상태를 확인한다.
- 전체 계정 또는 개별 계정 기준으로 작업을 본다.
- 오늘 해야 할 콘텐츠 작업을 확인한다.
- 콘텐츠 기획과 일정을 관리한다.
- 성과를 계정과 플랫폼 기준으로 확인한다.
- API 실패 또는 미지원 플랫폼은 보조 수기 입력으로 기록한다.

## UX 원칙

- 토큰, 계정 ID, API 상세 설정은 메인 탭에 노출하지 않는다.
- 성과 정보는 성과 탭에 모으고, 운영 작업과 섞지 않는다.
- 계정 필터는 주요 화면의 기준으로 일관되게 작동한다.
- 수기 입력은 보조 흐름으로만 배치한다.
- 색상, 폰트, 세부 스타일은 UI 시스템 단계에서 정한다.
## 콘텐츠 매니징 UX 방향

- 첫 화면은 “오늘 뭘 올렸고, 다음에 뭘 올릴지”를 바로 보여주는 운영 홈이어야 한다.
- 메인 탭의 첫 정보는 월간 캘린더이며, API로 가져온 실제 게시물이 업로드 완료 콘텐츠로 표시되는 흐름을 기본으로 한다.
- 성과 연결은 중심 흐름이 아니라 콘텐츠 관리 결과를 보조하는 분석 흐름이다.
- 기술 설정, 토큰, API 디버그 정보는 계정 설정 탭 안에서 접거나 숨긴다.
