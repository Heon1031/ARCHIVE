# 메인 탭 UX

## 목적

메인 탭은 창작 운영의 중심 화면이다. 사용자가 오늘 무엇을 해야 하는지 빠르게 확인하고 콘텐츠 일정을 관리한다.

## 기본 영역

- 상단 계정 필터
- 오늘 할 일
- 리마인드
- 업로드 예정 콘텐츠
- 콘텐츠 캘린더
- 콘텐츠 기획 목록
- 제작 상태 보드
- 보조 수기 입력 진입점

## 콘텐츠 캘린더

- 전체 계정 또는 선택 계정 기준으로 일정을 보여준다.
- 업로드 예정일, 플랫폼, 계정, 제작 상태를 함께 확인할 수 있어야 한다.
- 성과 분석 수치는 캘린더의 중심 정보로 두지 않는다.

## 콘텐츠 기획 목록

- 아이디어, 주제, 플랫폼, 대상 계정, 예정일을 확인한다.
- 제작 상태로 이어질 수 있어야 한다.
- 계정이 정해지지 않은 기획은 별도 상태로 표시한다.

## 제작 상태

- 기획, 제작 중, 검토, 예약 예정, 완료 같은 상태를 기준으로 본다.
- 상태 변경은 콘텐츠 운영 흐름에 집중한다.
- API 토큰이나 연결 설정은 이 영역에 넣지 않는다.

## 리마인드와 오늘 할 일

- 앱 첫 진입 시 가장 먼저 파악할 수 있어야 한다.
- 오늘 마감, 업로드 예정, 확인 필요한 API 실패 항목을 구분한다.
- 수기 입력은 API 실패 또는 미지원 채널 항목에서만 보조로 연결한다.

## 메인 탭 제외 요소

- Access Token 입력
- 계정 ID 상세 관리
- API 테스트 중심 화면
- 전체 성과 분석
- 플랫폼별 깊은 성과 비교
- 자동 업로드 설정
## 캘린더 우선 구조

- 메인 탭 첫 영역은 월간 콘텐츠 캘린더다.
- 캘린더에는 업로드 완료 게시물, 예정 콘텐츠, 보류 콘텐츠를 구분해 표시한다.
- API로 가져온 실제 게시물은 업로드 완료 상태의 `ContentItem`으로 자동 표시되는 방향을 기본으로 한다.
- 오늘 할 일, 리마인드, 기획 목록, 제작 상태 요약은 캘린더 아래 또는 보조 영역에 둔다.
- 콘텐츠 종류와 주제 분류는 캘린더/기획 UX와 연결한다.
## 매니저 캘린더 구조

- 메인 탭은 “그래서 오늘/이번 주에 뭘 할까?”에 답하는 운영 판단 화면이다.
- 화면은 좌측 운영 판단 패널과 우측 월간 달력으로 나눈다.
- 좌측에는 오늘의 방향, 리마인드, 추천, 개선, 휴식 판단, 이번 달 흐름 요약 카드를 둔다.
- 각 판단 카드는 클릭 시 왜 그렇게 판단했는지 짧은 근거를 보여준다.
- 우측 달력은 실제 게시물 배지와 날짜별 추천/개선/재활용/휴식 배지를 보여준다.
- 빈 날짜는 작성 폼이 아니라 운영 추천과 쉬어도 되는지 여부를 보여준다.
- 메인 탭에는 콘텐츠 작성/추가 폼을 노출하지 않는다.

## 달력 뱃지

- 실제 게시물 뱃지와 운영 판단 뱃지는 시각적으로 구분한다.
- Instagram/Threads 뱃지는 API로 가져온 실제 게시물을 의미한다.
- 추천/개선/재활용/휴식 뱃지는 운영 판단을 의미한다.
- 실제 게시물 뱃지를 클릭하면 게시물 상세와 성과 요약을 보여준다.
- 운영 판단 뱃지를 클릭하면 추천 이유, 기준 데이터, 주의점을 보여준다.
- 링크 열기 버튼은 메인 기본 UI에 두지 않는다.

## 날짜별 판단 기준

- 실제 게시물이 있는 날은 실제 게시물 뱃지를 우선한다.
- 실제 게시물 다음날은 휴식 또는 가벼운 추천을 우선한다.
- 2~3일 이상 비어 있으면 추천 뱃지를 배치한다.
- 30일 이상 지난 고반응 게시물이 있으면 재활용 후보로 보여준다.
- 성과가 낮은 형식이 반복되면 개선 후보로 보여준다.

## 레이아웃 토큰 원칙

- 메인 탭은 좌측 운영 판단, 우측 월간 달력의 2열 grid를 기본으로 한다.
- 캘린더 셀은 날짜 영역과 뱃지 영역을 분리해 날짜/뱃지 기준선이 흔들리지 않게 한다.
- 뱃지는 콘텐츠 종류나 판단만 짧게 표시하고, 긴 설명은 선택 상세에서 보여준다.
- 리마인드는 좌측 패널 하단에 제목/설명 박스와 콘텐츠 카드의 2단 구조로 둔다.
- 간격, 카드 크기, 캘린더 셀, 뱃지 높이, 리마인드 썸네일 크기는 CSS 변수로 관리한다.
- 임의의 개별 margin, width, height 추가는 피하고 기존 레이아웃 토큰을 우선 조정한다.
