# Layout Wireframes

이 문서는 메인탭 레이아웃이 계속 흔들리지 않도록 ASCII 와이어프레임과 CSS Grid 기준을 고정한다.

UI 수정 시 우선순위는 다음과 같다.

1. 이 문서의 레이아웃 구조
2. 기존 컴포넌트 구조
3. 세부 스타일

레이아웃을 바꿔야 한다면 먼저 이 문서를 수정하고, 그 다음 코드를 수정한다.

---

## 1. 메인탭 핵심 원칙

메인탭은 콘텐츠 작성 화면이 아니다.

메인탭의 역할:

- 월간 게시물 확인
- 날짜별 운영 추천 확인
- 오늘/이번 주 운영 판단 확인
- 다시 살릴 콘텐츠 확인

메인탭에 넣지 않는 것:

- 콘텐츠 작성 폼
- 콘텐츠 수동 추가 UI
- API 연결 UI
- DB 테이블
- 성과 상세 테이블

---

## 2. 가로모드 기본 레이아웃

가로모드에서는 달력과 운영 판단을 나란히 둔다.

비율 원칙:

- 달력은 화면의 주 영역이다.
- 운영 판단은 오른쪽 보조 영역이다.
- 리마인드는 운영 판단 안에 포함한다.
- 달력과 운영 판단은 정해진 비율을 유지한다.

```txt
┌────────────────────────────────────────────────────────────────────┐
│ Header / Tabs / Account Filter                                     │
├──────────────────────────────────────────────┬─────────────────────┤
│ Calendar Panel                                │ Operation Panel     │
│                                              │                     │
│ ┌──────────────────────────────────────────┐ │ ┌─────────────────┐ │
│ │ 월간 콘텐츠 캘린더                       │ │ │ 오늘의 방향     │ │
│ │                                          │ │ └─────────────────┘ │
│ │ 날짜 + IG/TH/짧은글/이미지/릴스 뱃지     │ │ ┌─────────────────┐ │
│ │                                          │ │ │ 이번 주 보강    │ │
│ │                                          │ │ │ 키워드          │ │
│ │                                          │ │ └─────────────────┘ │
│ │                                          │ │ ┌─────────────────┐ │
│ │                                          │ │ │ 개선 후보       │ │
│ │                                          │ │ └─────────────────┘ │
│ │                                          │ │ ┌─────────────────┐ │
│ │                                          │ │ │ 선택한 판단     │ │
│ │                                          │ │ │ 상세            │ │
│ │                                          │ │ └─────────────────┘ │
│ │                                          │ │ ┌─────────────────┐ │
│ │                                          │ │ │ 오늘의 리마인드 │ │
│ │                                          │ │ │ 썸네일 + 관련글 │ │
│ │                                          │ │ └─────────────────┘ │
│ └──────────────────────────────────────────┘ │                     │
└──────────────────────────────────────────────┴─────────────────────┘
```

권장 CSS 개념:

```css
.main-content-grid {
  display: grid;
  grid-template-columns: minmax(0, 1fr) minmax(320px, 380px);
  gap: var(--main-grid-gap);
  align-items: start;
}

.calendar-panel,
.operation-panel {
  min-width: 0;
}
```

금지:

```css
grid-template-columns: auto auto;
grid-template-columns: 1fr 1fr;
```

이유:

- `auto auto`는 콘텐츠 크기에 따라 화면을 밀어낸다.
- `1fr 1fr`은 운영 판단이 과하게 넓어지고 달력이 좁아진다.

## 3. 세로모드/좁은 화면 레이아웃

세로모드 또는 좁은 화면에서는 1열로 전환한다.

현재 제품에서는 달력을 먼저 보는 흐름이 핵심이므로 기본 순서는 다음과 같다.

```txt
┌──────────────────────────────┐
│ Header / Tabs / Filter       │
├──────────────────────────────┤
│ Calendar Panel               │
│ 월간 콘텐츠 캘린더           │
├──────────────────────────────┤
│ Operation Panel              │
│ - 오늘의 방향                │
│ - 이번 주 보강 키워드        │
│ - 개선 후보                  │
│ - 선택한 판단 상세           │
│ - 오늘의 리마인드            │
└──────────────────────────────┘
```

권장 CSS 개념:

```css
@media (max-width: 1024px) {
  .main-content-grid {
    grid-template-columns: 1fr;
  }

  .calendar-panel {
    order: 1;
  }

  .operation-panel {
    order: 2;
  }
}
```

## 4. 달력과 운영 판단 비율

가로모드 기준:

- 달력: 남는 공간 전체
- 운영 판단: 320px ~ 380px
- gap: 24px ~ 32px

권장:

```css
grid-template-columns: minmax(0, 1fr) minmax(320px, 380px);
```

운영 판단 폭이 380px을 넘어가면 달력 가독성이 떨어진다.
달력 폭이 너무 좁아지면 뱃지와 날짜 정렬이 깨진다.

---

## 5. Operation Panel 역할

Operation Panel은 보조 판단 영역이다.

포함:

- 오늘의 방향
- 이번 주 보강 키워드
- 개선 후보
- 선택한 판단 상세
- 오늘의 리마인드

포함하지 않음:

- 콘텐츠 추가 폼
- DB 목록
- API 연결 버튼
- 성과 그래프
- 긴 표

---

## 6. 리마인드 구조

리마인드는 Operation Panel 안에 포함한다.
하단 전체 폭으로 빼지 않는다.

```txt
┌──────────────────────────────┐
│ 오늘의 리마인드       썸네일 │
│ 오늘 다시 꺼내볼 후보입니다. │
├──────────────────────────────┤
│ 관련 게시물 제목             │
│ 캡션 2줄                     │
│ 주제 / 형식 / 반응점수       │
│ 추천 이유 1줄                │
│        ● ○ ○                 │
└──────────────────────────────┘
```

규칙:

- 썸네일은 작게 표시한다.
- 리마인드 이미지는 운영 패널 폭을 넘지 않는다.
- 제목은 1줄로 제한한다.
- 캡션은 2줄로 제한한다.
- 추천 이유는 1줄로 제한한다.
- 슬라이드 점 indicator는 하단에 둔다.
- 리마인드는 선택 날짜가 아니라 오늘 기준으로 유지한다.

## 7. 캘린더 셀 구조

```txt
┌──────────────┐
│ 13 오늘      │
│              │
│ [IG]         │
│ [이미지]     │
└──────────────┘
```

규칙:

- 날짜는 항상 좌상단에 둔다.
- 뱃지는 날짜 아래에 둔다.
- 뱃지는 최대 2개만 표시한다.
- 나머지는 `+N`으로 표시한다.
- 뱃지는 콘텐츠 종류만 짧게 표시한다.
- 긴 설명은 선택한 판단 상세에 표시한다.

뱃지 라벨:

- IG
- TH
- 짧은글
- 긴글
- 이미지
- 릴스
- 캐러셀
- 재활용
- 개선
- 휴식

금지:

- 추천-짧은글
- 추천-이미지
- 긴 문장형 뱃지

---

## 8. 금지 사항

- 리마인드 카드가 달력 폭을 밀어내는 구조 금지
- 왼쪽/오른쪽 패널에 `auto auto` 레이아웃 금지
- 날짜 셀 안에 긴 문장 삽입 금지
- 뱃지에 `추천-` 접두어 반복 금지
- 콘텐츠 추가/작성 UI를 메인탭에 재추가 금지
- 임의 margin으로 레이아웃 맞추기 금지
- CSS 변수 없이 숫자값을 여러 곳에 흩뿌리는 것 금지
- API 관련 로직을 레이아웃 수정과 함께 수정 금지

## 9. CSS 기준

메인 레이아웃은 아래 개념을 따른다.

```css
.main-content-grid {
  display: grid;
  grid-template-columns: minmax(0, 1fr) minmax(320px, 380px);
  gap: var(--main-grid-gap);
  align-items: start;
}

.calendar-panel,
.operation-panel {
  min-width: 0;
}
```

캘린더 셀은 아래 개념을 따른다.

```css
.calendar-cell {
  display: flex;
  flex-direction: column;
  align-items: flex-start;
}

.calendar-badge-stack {
  width: 100%;
  min-width: 0;
}
```

리마인드 이미지는 아래 개념을 따른다.

```css
.reminder-thumbnail {
  width: 100%;
  max-width: 100%;
  object-fit: cover;
  overflow: hidden;
}
```
