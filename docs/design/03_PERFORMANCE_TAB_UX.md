# 성과 탭 UX

## 목적

성과 탭은 API 기반 데이터를 바탕으로 전체, 플랫폼, 계정, 콘텐츠 단위의 성과를 확인하는 화면이다.

## 기본 영역

- 상단 계정/플랫폼 필터
- 전체 계정 성과 요약
- 플랫폼별 성과
- 개별 계정 성과
- 콘텐츠별 성과 목록
- API 데이터 상태
- 보조 수기 데이터 표시

## 전체 계정 성과

- 선택된 기간과 필터 기준의 합산 성과를 보여준다.
- Instagram과 Threads가 함께 선택된 경우 플랫폼별 구분이 가능해야 한다.

## 플랫폼별 성과

- Instagram 전체와 Threads 전체를 비교할 수 있어야 한다.
- 플랫폼마다 제공 가능한 지표가 다를 수 있음을 전제로 둔다.

## 개별 계정 성과

- 선택 계정의 성과를 별도로 확인한다.
- 여러 계정을 비교할 때는 계정명, 플랫폼, 주요 지표를 함께 보여준다.

## 콘텐츠별 성과

- 콘텐츠 단위로 조회수, 반응, 참여 지표를 확인한다.
- API 응답이 없는 콘텐츠는 상태를 분리해 보여준다.

## API 기반 성과 표시

- 로딩, 성공, 실패, 권한 부족, 토큰 만료 상태를 구분한다.
- 실패한 데이터는 조용히 숨기지 않는다.

## 수기 입력 데이터

- API 미지원 채널 또는 API 실패 케이스의 보조 데이터로만 표시한다.
- API 기반 성과와 같은 수준의 핵심 지표처럼 강조하지 않는다.
## 성과 탭 위치 조정

- 성과 탭은 수치만 보는 화면이 아니라 콘텐츠별 반응을 확인하는 보조 분석 화면이다.
- 콘텐츠 매니징의 중심은 메인 탭이며, 성과 탭은 “무엇이 반응을 얻었는지”를 확인하는 데 집중한다.
- 성과 탭에서 수동 연결을 강요하지 않고, API 동기화로 생성된 `ContentItem`과 `InsightRecord`의 관계를 우선 사용한다.
