# 수기 입력 보조 흐름

## 기본 원칙

수기 입력은 메인 기능이 아니다. API 우선 흐름을 보완하기 위한 예외 처리 기능이다.

## 사용 대상

- 브런치
- 블로그
- API 미지원 플랫폼
- API 실패 콘텐츠
- 일시적으로 데이터를 불러오지 못한 콘텐츠

## 위치할 수 있는 화면

- 메인 탭의 API 실패 또는 미지원 채널 항목
- 성과 탭의 API 데이터 없음 상태
- 계정 등록·설정 탭의 연결 실패 후 보조 기록 안내

## 입력 범위

- 플랫폼 또는 채널명
- 계정 또는 출처
- 콘텐츠 제목
- 날짜
- 최소 성과 수치
- 참고 메모

## 강조 제한 기준

- 메인 CTA로 배치하지 않는다.
- API 연결보다 먼저 보이지 않게 한다.
- 수기 데이터와 API 데이터를 같은 신뢰도로 표현하지 않는다.
- 수기 입력 항목에는 보조 데이터임을 표시한다.

## 제외 사항

- 수기 입력 기반 자동 성과 분석
- 수기 입력을 기본 콘텐츠 등록 방식으로 사용하는 흐름
- API 실패를 숨기기 위한 자동 대체 처리
