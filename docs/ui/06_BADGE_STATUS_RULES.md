# 뱃지와 상태 규칙

## 공통 원칙

- 뱃지는 플랫폼, 계정, 콘텐츠 상태, API 연결 상태를 빠르게 구분하는 용도로 사용한다.
- 색상은 상태 구분에 제한적으로 사용한다.
- 색상만으로 상태를 판단하게 하지 않는다.

## 플랫폼 뱃지

- Instagram과 Threads를 구분한다.
- 플랫폼 뱃지는 콘텐츠 카드, 계정 카드, 성과 목록에서 일관되게 사용한다.

## 계정 뱃지

- 개별 계정명을 짧게 표시한다.
- 전체 계정이나 플랫폼 전체 필터와 혼동되지 않게 한다.

## 콘텐츠 상태 뱃지

- 기획
- 제작 중
- 검토
- 예약 예정
- 완료
- 확인 필요

## API 연결 상태 뱃지

- 연결됨
- 확인 필요
- 실패
- 토큰 만료
- 권한 부족
- 동기화 중

## 상태 표시 기준

- 연결됨은 안정적인 상태로 표시한다.
- 실패, 토큰 만료, 권한 부족은 사용자가 조치를 알 수 있게 표시한다.
- 확인 필요는 오류와 정상 사이의 중간 상태로 다룬다.
- API 실패 상태를 숨기거나 성공처럼 보이게 하지 않는다.
