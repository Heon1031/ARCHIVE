# UI 디자인 규약 인덱스

이 폴더는 실제 UI 구현 전에 따라야 할 화면 구성 규칙과 UI 시스템 기준을 정리한다.

## 문서 목록

- `01_VISUAL_DIRECTION.md`: 전체 시각 방향과 피해야 할 디자인 톤
- `02_LAYOUT_RULES.md`: 레이아웃, 필터, 콘텐츠 영역 배치 규칙
- `03_TAB_NAVIGATION_RULES.md`: 메인/성과/계정 등록·설정 탭 규칙
- `04_CARD_RULES.md`: 콘텐츠, 계정, 성과 카드 규칙
- `05_BUTTON_RULES.md`: 버튼 우선순위와 액션 배치 규칙
- `06_BADGE_STATUS_RULES.md`: 플랫폼, 계정, 콘텐츠, API 상태 뱃지 규칙
- `07_FORM_RULES.md`: 계정/콘텐츠 폼과 민감 정보 입력 규칙
- `08_TABLE_AND_METRIC_RULES.md`: 성과 표와 지표 카드 규칙
- `09_EMPTY_ERROR_STATE_RULES.md`: 빈 상태와 오류 상태 안내 규칙

## 읽기 원칙

- 화면 레이아웃은 `02_LAYOUT_RULES.md`를 우선한다.
- 탭 구현은 `03_TAB_NAVIGATION_RULES.md`를 확인한다.
- 컴포넌트별 세부 규칙은 해당 문서만 읽는다.
