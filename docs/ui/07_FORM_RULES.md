# 폼 규칙

## 공통 원칙

- 폼은 입력 목적이 분명해야 한다.
- 필수 입력과 선택 입력을 구분한다.
- 저장, 취소, 테스트 액션의 위치를 일관되게 유지한다.

## 계정 등록 폼

- 플랫폼 선택을 먼저 배치한다.
- 계정 ID와 Access Token 입력을 분리한다.
- 연결 상태와 API 테스트 결과를 같은 흐름에서 확인할 수 있게 한다.

## 콘텐츠 등록 폼

- 제목, 대상 계정, 플랫폼, 예정일, 제작 상태를 우선 입력한다.
- 긴 메모나 보조 정보는 하단에 둔다.
- 수기 입력 콘텐츠는 보조 데이터임을 표시한다.

## 토큰 입력 필드

- 민감 정보임을 명확히 안내한다.
- 입력값 전체를 반복 노출하지 않는다.
- 다른 탭에서 토큰 값을 직접 표시하지 않는다.

## 계정 ID 입력 필드

- 플랫폼별로 필요한 계정 ID를 구분한다.
- 계정명과 계정 ID를 혼동하지 않도록 라벨을 명확히 한다.

## API 테스트 영역

- 계정 등록 폼 안에서 접근 가능해야 한다.
- 테스트 결과는 성공, 실패, 토큰 만료, 권한 부족으로 구분한다.
- 실패 시 다음 행동을 짧게 안내한다.

## 저장/취소 버튼

- 저장은 폼 하단 또는 고정된 액션 영역에 둔다.
- 취소는 저장보다 낮은 우선순위로 둔다.
- API 테스트는 저장과 별개의 액션으로 구분한다.
