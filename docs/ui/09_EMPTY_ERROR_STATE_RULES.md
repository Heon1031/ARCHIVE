# 빈 상태와 오류 상태 규칙

## 공통 원칙

- 사용자가 현재 상태와 다음 행동을 바로 알 수 있어야 한다.
- 오류를 숨기거나 성공처럼 표현하지 않는다.
- 안내 문구는 짧고 구체적으로 작성한다.

## 계정 미등록 상태

- 아직 등록된 계정이 없음을 알린다.
- 계정 등록·설정 탭에서 계정을 추가하도록 안내한다.
- 수기 입력을 첫 행동으로 유도하지 않는다.

## 콘텐츠 없음 상태

- 선택된 필터에 콘텐츠가 없음을 알린다.
- 콘텐츠 추가 또는 필터 변경을 다음 행동으로 제안한다.
- API 연결 문제인지 단순히 콘텐츠가 없는지 구분한다.

## API 실패 상태

- API 요청이 실패했음을 명확히 표시한다.
- 실패 원인, 재시도 가능 여부, 계정 설정 확인 동선을 제공한다.
- 필요할 때만 보조 수기 입력을 제안한다.

## 토큰 만료 상태

- 토큰 만료 또는 권한 문제를 구분한다.
- 계정 등록·설정 탭에서 토큰을 확인하도록 안내한다.
- 메인 탭에는 토큰 값 자체를 노출하지 않는다.

## 성과 데이터 없음 상태

- 아직 성과 데이터가 없거나 불러오지 못했는지 구분한다.
- API 기반 성과 불러오기 또는 계정 연결 확인을 안내한다.
- 수기 데이터가 있으면 보조 출처임을 표시한다.

## 문구 기준

- 문제 상황을 먼저 말한다.
- 다음 행동을 한 가지 이상 제안한다.
- 기술 오류 전문을 그대로 길게 노출하지 않는다.
