# 탭 내비게이션 규칙

## 탭 구성

- 메인
- 성과
- 계정 등록·설정

## 선택 상태

- 현재 선택된 탭은 다른 탭보다 명확히 구분한다.
- 선택 상태는 색상만으로 표현하지 않는다.
- 탭 전환 시 선택된 계정 필터 기준은 가능한 유지한다.

## 메인 탭 규칙

- 콘텐츠 캘린더, 기획, 제작 상태, 오늘 할 일을 중심으로 둔다.
- Access Token, 계정 ID 상세 설정, API 테스트 중심 UI를 넣지 않는다.

## 성과 탭 규칙

- 전체/플랫폼별/계정별/콘텐츠별 성과를 보여준다.
- 콘텐츠 제작 상태 관리나 계정 토큰 입력을 넣지 않는다.

## 계정 등록·설정 탭 규칙

- 계정 등록, 연결 상태, 마지막 동기화, API 테스트를 다룬다.
- 콘텐츠 캘린더나 성과 상세 분석을 넣지 않는다.

## 역할 혼선 방지

- 탭마다 하나의 주요 목적만 가져야 한다.
- 다른 탭의 기능은 요약 상태나 진입 링크 수준으로만 노출한다.
- 설정성 정보는 계정 등록·설정 탭에 모은다.
