# 카드 규칙

## 공통 원칙

- 카드는 반복되는 정보 묶음을 보여줄 때 사용한다.
- 카드 안에는 가장 중요한 정보부터 배치한다.
- 카드가 너무 많은 설명을 담지 않도록 한다.

## 콘텐츠 카드

### 우선 표시 정보

- 콘텐츠 제목
- 대상 계정
- 플랫폼
- 예정일
- 제작 상태
- 리마인드 여부

### 숨기거나 낮출 정보

- Access Token
- 계정 ID 상세값
- API 원본 응답
- 긴 메모 전문

## 계정 카드

### 우선 표시 정보

- 계정명
- 플랫폼
- 연결 상태
- 마지막 동기화
- API 테스트 액션

### 숨기거나 낮출 정보

- 토큰 전체값
- 내부 API 응답 전문
- 설정 세부 로그

## 성과 카드

### 우선 표시 정보

- 핵심 지표명
- 현재 값
- 기준 기간
- 적용 필터
- 데이터 출처

### 주의 사항

- 지표 카드를 과도하게 늘리지 않는다.
- API 데이터와 수기 입력 데이터의 출처를 구분한다.
- 큰 숫자만 강조하고 맥락을 숨기지 않는다.

## 밀도와 가독성

- 카드 제목, 상태, 날짜, 액션이 쉽게 구분되어야 한다.
- 반복 카드의 정보 순서는 일관되게 유지한다.
- 카드 안에 카드 형태를 중첩하지 않는다.
