# 버튼 규칙

## 버튼 우선순위

- 주요 액션: 현재 화면의 핵심 작업 하나
- 보조 액션: 필요한 부가 작업
- 위험 액션: 삭제, 연결 해제, 토큰 제거 같은 되돌리기 어려운 작업

## 주요 액션 위치

- 화면 또는 섹션 상단의 예측 가능한 위치에 둔다.
- 한 화면에 주요 액션을 과도하게 여러 개 두지 않는다.

## 보조 액션 위치

- 관련 카드, 행, 폼 영역 안에 배치한다.
- 주요 액션보다 낮은 시각 우선순위를 가진다.

## 위험 액션 규칙

- 일반 액션과 시각적으로 구분한다.
- 실수로 누르기 쉬운 위치에 두지 않는다.
- 확인 절차가 필요한 액션으로 다룬다.

## 주요 버튼별 기준

- 콘텐츠 추가: 메인 탭의 콘텐츠 기획 또는 캘린더 영역에서 주요 액션으로 사용한다.
- 계정 추가: 계정 등록·설정 탭에서 주요 액션으로 사용한다.
- API 연결 확인: 계정 카드 또는 계정 설정 폼의 보조 액션으로 사용한다.
- API로 성과 불러오기: 성과 탭의 보조 또는 섹션 액션으로 사용한다.
- 수기로 입력: API 실패, 미지원 채널 상황에서만 보조 액션으로 사용한다.

## 수기 입력 버튼

- 메인 CTA처럼 강조하지 않는다.
- API 연결 또는 API 재시도보다 먼저 보이지 않게 한다.
- 버튼 문구에서 보조 입력임을 알 수 있게 한다.
