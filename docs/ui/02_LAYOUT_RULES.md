# 레이아웃 규칙

## 전체 구조

- 상단에는 주요 탭을 둔다.
- 탭 아래에는 화면 기준이 되는 계정/플랫폼 필터를 둔다.
- 본문은 메인 콘텐츠 영역을 중심으로 구성한다.
- 보조 패널은 추가 맥락이나 낮은 우선순위 작업에만 사용한다.

## 상단 탭

- 메인, 성과, 계정 등록·설정 탭을 명확히 구분한다.
- 탭은 항상 같은 위치에서 작동해야 한다.
- 탭 이름은 기능 설명보다 역할 중심으로 짧게 유지한다.

## 계정 필터 위치

- 주요 화면의 상단 가까이에 둔다.
- 현재 선택 기준이 전체 계정, 플랫폼 전체, 개별 계정 중 무엇인지 분명히 보여준다.
- 필터는 콘텐츠 영역 안쪽 깊은 곳에 숨기지 않는다.

## 메인 콘텐츠 영역

- 메인 탭은 오늘 할 일과 콘텐츠 흐름을 먼저 배치한다.
- 성과 탭은 요약 지표와 세부 목록의 순서로 배치한다.
- 계정 등록·설정 탭은 계정 목록과 연결 관리 영역을 분리한다.

## 보조 패널 사용 기준

- 리마인드, API 실패 알림, 보조 수기 입력 진입점에 사용한다.
- 핵심 작업을 보조 패널 안에 숨기지 않는다.
- 보조 패널이 메인 작업보다 커 보이면 안 된다.

## 정보 밀도

- 메인 탭은 일정과 작업이 한눈에 보이도록 중간 밀도를 유지한다.
- 성과 탭은 핵심 지표를 먼저 보여주고 세부 데이터는 표로 정리한다.
- 계정 설정 탭은 폼과 상태 정보를 명확히 나누어 읽기 쉽게 한다.
