# 공통 작업 규칙

모든 에이전트는 이 문서를 우선으로 따른다.

## 기본 원칙

- 요청받은 범위만 최소 변경한다.
- 전체 리팩터링, 대규모 구조 변경, 불필요한 파일 이동을 하지 않는다.
- 기존 코드 스타일과 폴더 구조를 우선한다.
- 한 번에 하나의 목적을 가진 변경만 수행한다.
- 사용자가 명시하지 않은 기능을 임의로 추가하지 않는다.

## 토큰 절약 규칙

- 필요한 파일만 읽고 작업한다.
- 전체 코드나 전체 문서를 재출력하지 않는다.
- 긴 설명 대신 변경 이유와 핵심 영향만 요약한다.
- 동일한 제품 설명은 `02_PRODUCT_CONTEXT.md`를 참조하고 반복하지 않는다.

## 금지 사항

- 요청 범위를 넘어 앱 코드를 수정하지 않는다.
- 패키지 설치, 설정 변경, 빌드 도구 변경은 명시 요청이 있을 때만 한다.
- Access Token, API Key, Secret 값을 코드나 문서에 노출하지 않는다.
- 임시 구현을 실제 기능처럼 보이게 만들지 않는다.
- 자동 업로드, OAuth 전체 플로우, 데이터 모델을 임의로 확장하지 않는다.

## 작업 후 출력 형식

작업 완료 후 다음 순서로 짧게 보고한다.

1. 변경 요약
2. 수정 또는 생성 파일 목록
3. 확인한 내용
4. 남은 이슈 또는 확인 필요 사항
