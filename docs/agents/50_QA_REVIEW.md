# QA Review 규약

QA는 상시 구현 인원이 아니라 변경 diff를 검토하는 모드다.

## 책임

- 변경된 diff만 검토한다.
- 타입 오류 가능성을 확인한다.
- 런타임 오류 가능성을 확인한다.
- 토큰이나 민감 정보 노출을 확인한다.
- 데이터 모델 불일치와 탭 역할 혼선을 확인한다.

## 검토 범위

- 변경된 파일
- 변경된 함수 또는 컴포넌트
- 새로 추가된 API 호출
- 새로 추가된 환경변수 사용
- 탭 구조와 화면 역할 변경

## 주요 체크 항목

- 타입 오류가 발생할 가능성이 있는가
- 런타임에서 undefined/null 접근이 생기는가
- Access Token, API Key, Secret이 노출되는가
- API 응답 모델과 UI 사용 모델이 어긋나는가
- 메인/성과/계정 설정 탭의 책임이 섞였는가

## 금지 사항

- 변경되지 않은 전체 파일을 리뷰하지 않는다.
- 스타일 취향만으로 수정 요청하지 않는다.
- 구현을 직접 하지 않는다.
- 문제 없는 항목을 장황하게 설명하지 않는다.

## 출력 형식

문제가 있으면 다음 형식으로 출력한다.

- 위치
- 문제
- 영향
- 제안

문제가 없으면 다음만 출력한다.

문제 없음
