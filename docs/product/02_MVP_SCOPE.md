# MVP 범위

## 반드시 만들 기능

- 여러 Instagram / Threads 계정 등록
- 계정 ID와 Access Token 입력
- 계정별 연결 상태 표시
- 마지막 동기화 시각 표시
- API 테스트 실행 흐름
- 전체 계정/개별 계정 필터
- 콘텐츠 캘린더와 기획 목록
- 콘텐츠 제작 상태 관리
- 오늘 할 일과 리마인드
- API 기반 성과 데이터 표시를 위한 기본 구조

## 있으면 좋은 기능

- 플랫폼별 빠른 필터
- 콘텐츠별 간단 메모
- API 실패 시 보조 수기 기록
- 계정별 운영 목적 태그
- 성과 지표의 간단한 증감 표시

## MVP에서 제외할 기능

- 자동 업로드
- 상용 OAuth 전체 플로우
- 로그인
- 결제
- 팀 협업
- Google Calendar / Google Sheets 연동
- 고급 AI 추천 시스템

## 1차 완성 기준

- 사용자가 여러 계정을 등록하고 연결 상태를 확인할 수 있다.
- 전체 계정 또는 개별 계정 기준으로 콘텐츠 운영 화면을 볼 수 있다.
- 메인/성과/계정 등록·설정 탭의 역할이 분리되어 있다.
- API 테스트와 API 기반 성과 표시를 연결할 수 있는 구조가 준비되어 있다.
- 수기 입력은 보조 흐름으로만 접근 가능하다.
## 콘텐츠 매니징 기준 보정

- MVP의 중심은 API 기반 게시물을 캘린더에 반영하는 콘텐츠 운영 흐름이다.
- API로 가져온 게시물은 `ContentItem`으로 자동 생성 또는 업데이트될 수 있어야 한다.
- 게시물 본문, 캡션, 텍스트를 바탕으로 콘텐츠 종류와 주제를 기록할 수 있는 구조를 준비한다.
- 성과 수치는 계속 `InsightRecord`에 분리하고, `ContentItem`에는 직접 넣지 않는다.
