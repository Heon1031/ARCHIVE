# 제품 방향

## 제품 정체성

여러 Instagram / Threads 계정을 운영하는 개인 창작자를 위한 콘텐츠 운영 대시보드다.

## 대상 사용자

- 여러 SNS 계정을 혼자 운영하는 개인 창작자
- 콘텐츠 기획, 제작, 발행 준비, 성과 확인을 한 화면 흐름에서 관리하려는 사용자
- Instagram / Threads 성과를 계정별로 비교하고 싶은 사용자

## 핵심 문제

- 여러 계정의 콘텐츠 일정과 제작 상태가 흩어져 있다.
- 플랫폼별 성과와 계정별 성과를 함께 보기 어렵다.
- API 연결 상태, 토큰, 계정 ID 관리가 운영 화면과 분리되어 있지 않다.
- API 미지원 채널이나 실패 케이스는 별도 보조 기록이 필요하다.

## 해결 방향

- 계정 등록과 API 연결 상태를 먼저 안정적으로 다룬다.
- 전체 계정 보기와 개별 계정 보기를 모두 지원한다.
- 메인 탭은 콘텐츠 운영, 성과 탭은 분석, 계정 등록/설정 탭은 연결 관리에 집중한다.
- 수기 입력은 API 미지원 채널과 API 실패 케이스를 위한 보조 기능으로 제한한다.

## API 우선 원칙

- Instagram / Threads API 연결을 제품의 1차 데이터 흐름으로 둔다.
- 콘텐츠와 성과 데이터는 가능한 한 API에서 불러온다.
- API 실패 상태는 숨기지 않고 사용자에게 연결 상태와 오류 원인을 보여준다.

## 수기 입력 원칙

- 수기 입력은 메인 기능이 아니다.
- 브런치, 블로그, API 미지원 플랫폼, API 실패 케이스를 기록하는 보조 기능이다.
- 수기 입력이 API 기반 계정 운영 흐름보다 먼저 보이거나 더 강조되면 안 된다.
## 콘텐츠 매니징 캘린더 정렬

- 제품의 핵심 정체성은 **Instagram / Threads 멀티 계정 콘텐츠 매니징 캘린더**다.
- 사용자는 성과 연결만 하는 것이 아니라 “무엇을 올렸고, 어떤 주제가 많았고, 다음에 무엇을 올릴지”를 확인해야 한다.
- API로 가져온 실제 게시물은 업로드 완료 콘텐츠로 자동 기록되어 메인 캘린더에 표시되는 방향을 기본으로 둔다.
- 기존 콘텐츠에 API 성과를 사용자가 직접 연결하는 흐름은 예외 처리 또는 수동 보정 흐름으로 낮춘다.
